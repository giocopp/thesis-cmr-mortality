# IOM Data Cleaning Pipeline

This directory contains raw IOM data files and a cleaning script to process them into CSV files.

## Data Sources

1. **ALL MED DATA 2010-2025_12.08.2025.xlsx**
   - Mediterranean crossings monthly time series data
   - Coverage: 2014-2025
   - Issue: Years 2024 and 2025 lack year markers in the Excel structure

2. **IOM_MMP data_2014-2025_12.08.2025.xlsx**
   - IOM Missing Migrants Project incident data
   - Coverage: 2014-2025
   - Separate sheets for each year plus index


The script will:

2. Process and clean the data
3. Validate data integrity
4. Output cleaned CSV files to the `Clean/` directory

### Output Files

The script generates the following clean CSV files in the `Clean/` directory:

1. **med_crossings_monthlyTS.csv**
   - Monthly Mediterranean crossings time series
   - 144 rows (12 years × 12 months)
   - Years: 2014-2025
   - Fixed year labels for 2024 and 2025 data

2. **iom_mmp_2014_2025_all_types.csv**

4. **iom_mmp_index.csv**
   - Reference index with category definitions
   - Includes source quality levels, cause of death categories, etc.

## Known Data Issues (Now Fixed)

### Mediterranean Crossings Data

**Problem:** The original Excel file has an unusual structure where:
- 2023 data has a year marker in the first column
- 2024 and 2025 data do NOT have year markers
- This caused all 2024 and 2025 data to be incorrectly labeled as "2023"

**Solution:** The cleaning script uses Excel row boundaries to correctly assign years:
- Rows 120-131: 2023 (12 months)
- Rows 133-144: 2024 (12 months)
- Rows 146-157: 2025 (12 months, Nov-Dec are NaN)

**Validation:** The script validates that:
- Each year has exactly 12 month records
- No duplicate year-month combinations exist
- All expected years (2014-2025) are present
